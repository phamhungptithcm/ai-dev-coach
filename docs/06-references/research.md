# Research Notes

This product direction is based on practical observations in early-career developer workflows:

- speed gains from AI can hide weak reasoning loops
- copy-paste convenience increases dependency risk
- structured prompting improves answer quality and learning retention

## Working Hypothesis

Developers learn faster with AI when they are nudged to include:

- explicit context
- prior attempts
- constraints and tradeoffs

## Validation Plan

- run user interviews with beginners and mentors
- compare baseline prompting vs guided prompt-builder prompting
- measure perceived learning confidence and debugging quality
