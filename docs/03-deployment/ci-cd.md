# CI/CD Pipeline

## Workflows

- `ci.yml`
Runs on pull requests and branch pushes (`staging`, `main`) to validate extension code and docs.

- `deploy-docs.yml`
Runs on `staging` to build and deploy MkDocs docs to GitHub Pages.

- `release.yml`
Runs on `main` to auto-create release tag from `extension/manifest.json`, generate GitHub release notes, package extension, and publish to Chrome Web Store when required secrets are configured.

## CI Validation Gates

`ci.yml` enforces:

1. `node --check` on all extension JavaScript files
2. JSON validation for `extension/manifest.json`
3. extension package integrity validation (`node scripts/validate-extension.mjs`)
4. junk-file rejection (`.DS_Store`)
5. strict MkDocs build (`mkdocs build --strict`)

## Chrome Web Store Publish Flow

1. Merge cycle promotes code from `staging` to `main`.
2. Release workflow reads `extension/manifest.json` version.
3. If tag `v<version>` does not exist, workflow creates and pushes it.
4. Workflow creates/updates GitHub Release with autogenerated notes.
5. Workflow packages extension zip in `dist/`.
6. Workflow uploads extension package to Chrome Web Store API.
7. Workflow publishes uploaded package (`default` target unless overridden by `CWS_PUBLISH_TARGET`).
